Description

Message segments are defined in hl7 v2 specification. They contain fields, and each field outside of special cases, contains 1 field repetition. The crud operations make sure to handle this. Special cases, such as MSH.1 and MSH.2, do not have a field repetition, while MSH.5 does.

The structure of message segments is as follows

Field -> FieldRepetition -> Components -> Sub Components

Properties

  • SegmentName: The name of the segment.
  • Fields: The fields within the segment.

Methods

  • ToString: Converts the message segment to a string using the Values of it's fields.
  • Rebuild: Rebuilds the values of each of the message segments fields.
  • GetField: Gets a field by Id.
  • AddField: Adds a field to end of the message segment.
  • RemoveField: Removes a field by Id from the message segment.
  • InsertField: Inserts a field by Id into the message segment.
  • UpdateField: Updates a field by Id in the message segment.

An indexer is exposed to access fields by their Id.
